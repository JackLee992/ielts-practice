## Listening Drill

### Set

Research around the Chembe Bird Sanctuary, Part 4 note completion, questions 31-40

### Raw Result

- Score: 3/10

- Wrong: 31, 32, 33, 34, 36, 39, 40

### What Broke

The main problem was not basic comprehension. The real bottleneck was answer selection inside the note-completion structure:

- category words were missed
- summary/result words were replaced by examples
- some exact words were understood but not written
- spelling still leaked marks

### Key Error Pattern

#### Category or class word missed

- 31: `rats and other rodents`
- the answer had to be a type of rodent, not `birds`

#### Result or summary word replaced by an example

- 39: the answer was `noise`
- `pan` was only one example of how the noise was made

#### Direct content not written down

- 33: `tourism`
- 34: `traffic`

#### Spelling leak

- 36: `poison`
- 40: `combination`

### Listening Takeaway

This set confirmed a useful Part 4 rule:

1. before writing, decide whether the blank needs a category, result, cause, or example
2. if the text gives an example, do not automatically copy it into the blank
3. the last seconds should be used to scan spelling, especially nouns

## Reading Drill

Advanced driving course, questions 8-14

### Result

- Score: 6/7

### What Broke

The wrong answer did not come from a major logic failure. The problem was over-weighting a small modifier:

- statement: `on the road ahead`
- text: `when you are driving`

The real match was in the main idea:

- `predict problems`
- `anticipate hazards`

### Takeaway

- compare the main idea first
- do not reject a `TRUE` answer because of a small modifier difference

## Listening Training Change To Trial

From this point, listening practice will use a more fixed review workflow for one week.

Reason:

- reading review felt sticky because the error was being turned into one reusable rule
- listening review still felt less concrete after the set ended
- replaying only wrong questions feels more useful than immediate full-set repetition

### Trial workflow

1. one set = one main listening target
2. after the set, write only:
   - `This set trained`
   - `Wrong questions`
   - `Main trap`
   - `Saved rule`
3. same-day replay = wrong or unsure questions only
4. next-day test = use a new short set to check whether the rule transfers

### Trial evaluation point

Check after roughly one week whether:

1. listening mistakes feel easier to explain
2. the saved rule can be recalled before the next set
3. the same trap appears less often in new material
